# Phase 2: Report Pipeline

Create a script that runs BDD tests and generates both an HTML report (rich, interactive) and a markdown summary (GitHub-renderable). Both reports are committed to the repo.

## Context — read these files first

| # | File | What to extract |
|---|------|-----------------|
| 1 | `docs/docs.conventions.md` | Generated content markers (`<!-- BEGIN GENERATED -->` / `<!-- END GENERATED -->`) |
| 2 | `docs/standards/delivery/local-development.md` | Current build/run/test instructions — add report generation |
| 3 | `src/tests/CompileAndSip.Bdd.Tests/CompileAndSip.Bdd.Tests.csproj` | Test project reference and package versions |
| 4 | `docs/business/scenarios/scenarios.index.md` | Scenario IDs and names to map in the report |
| 5 | `docs/docs.why-what-how.bdd.md` | Verification reporting section (written in Phase 1) for alignment |

## Steps

### 1. Create `docs/business/verification/` directory

This will hold the generated verification reports.

### 2. Create `src/generate-bdd-report.sh`

The script should:

1. **Run BDD tests** from the `src/` directory with both TRX and HTML loggers:
   ```bash
   dotnet test tests/CompileAndSip.Bdd.Tests \
     --logger "trx;LogFileName=bdd-results.trx" \
     --logger "html;LogFileName=bdd-results.html" \
     --results-directory ./test-results
   ```

2. **Copy the HTML report** to `docs/business/verification/bdd-results.html`

3. **Parse the TRX file** (XML) to extract:
   - Total tests, passed, failed, skipped
   - Per-test-class results (each class maps to a feature/scenario: `MenuSteps` → SCN-001, etc.)
   - Execution timestamp

   Use available tools (`xmllint`, `grep`, `sed`, `awk`, or `node` if `xmllint` is unavailable). The script should work in the devcontainer environment.

4. **Generate `docs/business/verification/bdd-status.md`** with this structure:

   ```markdown
   # BDD Verification Status

   <!-- BEGIN GENERATED: bdd-status -->

   > Generated by `src/generate-bdd-report.sh` — do not edit manually.
   > Last run: YYYY-MM-DD HH:MM UTC

   ## Summary

   | Result | Count |
   |--------|-------|
   | Passed | NN |
   | Failed | NN |
   | Skipped | NN |
   | **Total** | **NN** |

   ## Scenario Results

   | Scenario | Feature File | Tests | Status |
   |----------|-------------|-------|--------|
   | SCN-001 — Browse Menu and Select Drink | `SCN-001-browse-menu-select-drink.feature` | N passed | ✅ Pass |
   | SCN-002 — Customise Drink | `SCN-002-customise-drink.feature` | N passed | ✅ Pass |
   | SCN-003 — Payment Succeeds | `SCN-003-payment-succeeds.feature` | N passed | ✅ Pass |
   | SCN-004 — Payment Fails | `SCN-004-payment-fails.feature` | N passed | ✅ Pass |
   | SCN-005 — Kitchen Marks Order Complete | `SCN-005-kitchen-marks-order-complete.feature` | N passed | ✅ Pass |

   ## Detailed Report

   For a richer interactive view, open [bdd-results.html](bdd-results.html) in a browser.

   <!-- END GENERATED -->
   ```

   The scenario mapping can use the test class names or feature file names from the TRX output. The script should map them to the known SCN-xxx IDs.

5. **Make the script executable**: `chmod +x src/generate-bdd-report.sh`

### 3. Run the script and verify output

```bash
cd src
./generate-bdd-report.sh
cd ..
```

Check that both files exist and have correct content.

### 4. Update `docs/standards/delivery/local-development.md`

Add a section for generating verification reports:

```markdown
## Verification reports

Generate BDD verification reports (HTML + markdown summary):

```bash
cd src
./generate-bdd-report.sh
```

This produces:
- `docs/business/verification/bdd-status.md` — markdown summary viewable on GitHub
- `docs/business/verification/bdd-results.html` — interactive HTML report viewable in a browser

Both files are committed to the repo. Re-run the script after any test changes to update them.
```

### 5. Clean up test-results directory

Add `src/test-results/` to `.gitignore` — the intermediate TRX and HTML files in the working directory are not needed in the repo (only the copies in `docs/business/verification/` are committed).

## Verification

```bash
# Script exists and is executable
test -x src/generate-bdd-report.sh && echo "OK" || echo "FAIL"

# Both report files exist
test -f docs/business/verification/bdd-status.md && echo "OK" || echo "FAIL"
test -f docs/business/verification/bdd-results.html && echo "OK" || echo "FAIL"

# Markdown has generated content markers

# Should return 1

grep -c 'END GENERATED' docs/business/verification/bdd-status.md

# Markdown contains the scenario table
grep -c 'SCN-00' docs/business/verification/bdd-status.md
# Should return 5

# All tests still pass
cd src && dotnet test && cd ..
```
