# Scope

## In scope

| Category | What's included |
|----------|----------------|
| **Ordering** | Browse 3-drink menu, select a drink, customise (milk, toppings, sugar, ice, temperature), review order, confirm |
| **Payment** | Card payment via external payment provider, success and failure handling |
| **Kitchen display** | View active orders with full details, mark orders as complete |
| **Order lifecycle** | Order created → payment processed → sent to kitchen → marked complete |
| **Order identification** | Simple sequential order number per session, displayed to customer and on kitchen display |

## Out of scope

| What's excluded | Why |
|----------------|-----|
| **Customer accounts** — no login, no profiles, no order history | Simplicity. Every transaction is anonymous. |
| **Loyalty program** — no points, no rewards | Vision item, not MVP |
| **Mobile ordering** — no app, no web ordering | Vision item, not MVP |
| **Analytics** — no sales reports, no dashboards | Vision item, not MVP |
| **Inventory/stock** — no stock tracking, items are always available | Simplicity |
| **Multiple kiosks** — single kiosk instance | Simplicity |
| **Staff management** — no login for kitchen staff, no shift tracking | Simplicity |
| **Receipt printing** — no physical or digital receipts | Simplicity |
| **Multi-location** — single shop only | Vision item, not MVP |
| **Accessibility** — beyond basic usability | Simplicity for the reference example |
